# Profile Genie - Repository Summary

## Overview

Profile Genie is a modern profile management application built with .NET 9.0, featuring a Blazor Server frontend, ASP.NET Core Web API backend, and .NET Aspire orchestration. This repository contains the complete source code, tests, and infrastructure configuration.

## Technology Stack

| Layer | Technology |
|-------|------------|
| Runtime | .NET 9.0 |
| Frontend | Blazor Server |
| Backend | ASP.NET Core Web API |
| ORM | Entity Framework Core |
| Database | PostgreSQL |
| Orchestration | .NET Aspire |
| Testing | xUnit, FluentAssertions, Moq, bUnit, Playwright |
| Containerization | Docker |

## Key Components

### ProfileGenie.Core
- **Purpose**: Domain layer containing entities, interfaces, and business logic
- **Key Files**:
  - `Entities/Profile.cs` - Core domain entity
  - `Interfaces/IProfileService.cs` - Service contract
  - `Services/ProfileService.cs` - Service implementation

### ProfileGenie.Api
- **Purpose**: REST API layer exposing endpoints
- **Key Files**:
  - `Program.cs` - Application entry point
  - `Controllers/ProfilesController.cs` - API endpoints
  - `appsettings.json` - Configuration

### ProfileGenie.Frontend
- **Purpose**: Blazor Server UI
- **Key Files**:
  - `Program.cs` - Application entry point
  - `Components/` - Blazor components
  - `wwwroot/` - Static assets

### ProfileGenie.Shared
- **Purpose**: Shared DTOs and contracts between frontend and backend
- **Key Files**:
  - `DTOs/CreateProfileDto.cs` - Input DTO
  - `DTOs/ProfileDto.cs` - Output DTO

### ProfileGenie.ServiceDefaults
- **Purpose**: Common Aspire service configuration
- **Key Files**:
  - `Extensions.cs` - Service defaults extension methods

### ProfileGenie.AppHost
- **Purpose**: Aspire orchestration host
- **Key Files**:
  - `Program.cs` - Orchestration configuration

## Development Workflow

1. **Prerequisites**: .NET 9.0 SDK, PostgreSQL
2. **Build**: `dotnet build ProfileGenie/ProfileGenie.sln`
3. **Test**: `dotnet test ProfileGenie/ProfileGenie.sln`
4. **Run API**: `dotnet run --project ProfileGenie/src/ProfileGenie.Api`
5. **Run Frontend**: `dotnet run --project ProfileGenie/src/ProfileGenie.Frontend`

## CI/CD Pipeline

- **Triggers**: Push to main/feature branches, PRs
